# 웹 버전에서 에러가 날 수 있는 설정 및 기능 리스트

> **이 문서는 비개발자도 이해할 수 있도록 쉽게 작성되었습니다.**

## 📌 핵심 개념 이해하기

**로컬 버전 vs 웹 버전의 차이:**
- **로컬 버전**: 여러분의 컴퓨터에서 직접 실행되는 프로그램 (예: 설치형 앱)
- **웹 버전**: 인터넷의 서버에서 실행되고, 브라우저로 접속하는 프로그램 (예: 웹사이트)

**왜 문제가 생기나요?**
- 로컬 버전은 여러분의 컴퓨터를 직접 제어할 수 있지만
- 웹 버전은 서버에서 실행되므로 여러분의 컴퓨터를 직접 제어할 수 없습니다
- 마치 "원격으로 친구 컴퓨터의 프로그램을 실행하려고 하는 것"과 같습니다

---

## 1. 🖥️ 에디터 설정 관련 문제

### 문제 1: "에디터에서 열기" 버튼이 작동하지 않음

**무슨 문제인가요?**
- 사용자가 "에디터에서 열기" 버튼을 클릭하면
- 프로그램이 여러분 컴퓨터의 VS Code, Cursor 같은 에디터를 열려고 시도합니다
- 하지만 웹 버전은 서버에서 실행되므로 여러분의 컴퓨터를 직접 제어할 수 없어서 실패합니다

**어떤 상황에서 발생하나요?**
- 작업 화면에서 "에디터에서 열기" 버튼 클릭
- 프로젝트 화면에서 "에디터에서 열기" 버튼 클릭
- VS Code, Cursor, Windsurf, IntelliJ, Zed, Xcode 등 모든 에디터에서 발생

**사용자가 보게 되는 것:**
- 버튼을 눌렀는데 아무 일도 일어나지 않거나
- "에디터를 열 수 없습니다" 같은 에러 메시지가 나타남

**해결 방법:**
- ✅ **Remote SSH 설정을 사용**: 원격 서버에 SSH로 연결하는 방식으로 에디터를 열 수 있습니다
- ❌ **로컬 에디터 열기 비활성화**: Remote SSH가 설정되지 않았으면 버튼 자체를 비활성화하거나 경고 메시지 표시

---

### 문제 2: 커스텀 에디터 명령어가 작동하지 않음

**무슨 문제인가요?**
- 설정에서 "사용자 정의 에디터 명령"에 `code`, `subl`, `vim` 같은 명령어를 입력할 수 있습니다
- 웹 버전에서는 서버가 여러분의 컴퓨터에 있는 이런 명령어를 실행할 수 없습니다

**사용자가 보게 되는 것:**
- 설정은 저장되지만, 실제로 에디터를 열 때 에러가 발생

**해결 방법:**
- 웹 버전에서는 이 설정을 아예 보이지 않게 하거나
- Remote SSH 설정만 사용하도록 안내

---

### ✅ 정상 작동: Remote SSH 설정

**이건 괜찮습니다!**
- Remote SSH 호스트를 설정하면
- 에디터를 여는 대신 특별한 URL을 생성합니다
- 여러분의 브라우저가 이 URL을 열면, 여러분 컴퓨터의 에디터가 자동으로 원격 서버에 연결됩니다
- 이 방식은 웹 버전에서도 완벽하게 작동합니다!

## 2. 🤖 에이전트 설정 관련 문제

### 문제: Claude Code 외 다른 에이전트를 선택하면 작동하지 않음

**무슨 문제인가요?**
- 설정에서 "기본 에이전트 구성"을 선택할 때 여러 옵션이 보입니다:
  - CLAUDE_CODE ✅ (이건 작동함)
  - AMP ❌
  - GEMINI ❌
  - CODEX ❌
  - OPENCODE ❌
  - QWEN_CODE ❌
  - CURSOR_AGENT ❌
  - COPILOT ❌

**왜 문제가 되나요?**
- 이 에이전트들은 모두 여러분의 컴퓨터에서 직접 실행되는 프로그램입니다
- 웹 버전에서는 서버가 여러분의 컴퓨터에 있는 이런 프로그램을 실행할 수 없습니다
- 마치 "원격으로 친구 컴퓨터의 게임을 실행하려고 하는 것"과 같습니다

**사용자가 보게 되는 것:**
- 설정에서 다른 에이전트를 선택해도 저장은 됩니다
- 하지만 실제로 작업을 시작하면:
  - 에이전트가 실행되지 않거나
  - "에이전트를 찾을 수 없습니다" 같은 에러가 발생

**해결 방법:**
- 웹 버전에서는 **CLAUDE_CODE만 선택 가능**하도록 드롭다운을 제한하거나
- 다른 에이전트를 선택하면 "웹 버전에서는 Claude Code만 사용 가능합니다" 같은 경고 메시지 표시

## 3. 🌐 브라우저 자동 열기 문제

### 문제: 서버 시작 시 브라우저가 자동으로 열리지 않음 (또는 에러 발생)

**무슨 문제인가요?**
- 로컬 버전에서는 프로그램이 시작되면 자동으로 브라우저를 열어줍니다
- 웹 버전에서는 서버가 클라우드에 있으므로 여러분의 컴퓨터 브라우저를 열 수 없습니다

**사용자가 보게 되는 것:**
- 서버가 시작될 때 브라우저를 열려고 시도하지만 실패하거나
- 불필요한 에러 메시지가 나타날 수 있습니다

**해결 방법:**
- 웹 버전에서는 브라우저 자동 열기 기능을 완전히 비활성화
- 사용자가 직접 브라우저에서 URL을 입력해서 접속하도록 안내

## 4. 📁 프로젝트 생성 관련 주의사항

### 주의: GitHub에서 프로젝트 생성 시 경로 문제 가능성

**무슨 문제인가요?**
- GitHub에서 프로젝트를 가져올 때 파일을 저장할 위치를 결정해야 합니다
- 로컬 버전은 여러분의 컴퓨터 경로를 사용하지만
- 웹 버전은 서버의 경로를 사용해야 합니다

**현재 상태:**
- 대부분의 경우 정상 작동할 수 있지만
- 경로 설정이 명확하지 않으면 문제가 발생할 수 있습니다

**확인 필요:**
- 웹 버전에서 프로젝트를 생성할 때 경로가 올바르게 설정되는지 테스트 필요

- 로컬 버전은 여러분의 컴퓨터에 파일을 저장합니다
- 웹 버전은 서버에 파일을 저장합니다

**확인 필요:**

## 📋 요약: 수정이 필요한 항목

### 🔴 즉시 수정 필요 (사용자가 직접 경험할 수 있는 문제)

1. **에디터 설정**
   - Remote SSH가 없을 때 "에디터에서 열기" 버튼이 작동하지 않음
   - → 버튼을 비활성화하거나 경고 메시지 표시 필요

2. **에이전트 설정**
   - Claude Code 외 다른 에이전트 선택 시 작동하지 않음
   - → 웹 버전에서는 Claude Code만 선택 가능하도록 제한 필요

3. **브라우저 자동 열기**
   - 서버 시작 시 불필요한 에러 발생 가능
   - → 웹 버전에서는 완전히 비활성화 필요

### 🟡 확인 필요 (잠재적 문제)

1. **프로젝트 경로**
   - GitHub에서 프로젝트 생성 시 경로가 올바른지 확인 필요

2. **파일 시스템 접근**
   - 모든 파일 저장/읽기 기능이 서버 경로를 올바르게 사용하는지 확인 필요

### ✅ 이미 완료됨

1. GitHub 통합 - 웹 버전에서만 표시
2. Claude 인증 - 웹 버전에서만 표시

## 💡 사용자 관점에서의 요약

**웹 버전 사용자가 겪을 수 있는 문제:**

1. ❌ "에디터에서 열기" 버튼을 눌렀는데 아무 일도 안 일어남
2. ❌ 다른 에이전트를 선택했는데 작업이 시작되지 않음
3. ⚠️ 서버 시작 시 불필요한 에러 메시지

**해결책:**

1. ✅ Remote SSH 설정을 사용하면 에디터 열기 가능
2. ✅ Claude Code만 사용하도록 안내
3. ✅ 브라우저 자동 열기 기능 제거

**결론:**
웹 버전에서는 "여러분의 컴퓨터를 직접 제어하는 기능"들이 작동하지 않습니다. 이런 기능들을 웹 버전에서는 비활성화하거나, 대안(Remote SSH 등)을 제공해야 합니다.
